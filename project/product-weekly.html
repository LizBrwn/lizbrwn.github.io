Why it picks well

聪明之处:
不被票数带着走。

直接把上百个产品丢给 AI,它只会选出热门大厂。所以这里用了两层选品——先用规则保证候选池里有黑马、有小众,再让模型在一个干净、均衡的池子里做最终判断、并写清理由。

评分维度、Prompt 版本、运行产物与评测结果都会随每次实验留痕。改进不凭感觉，而是回到黄金集、漏斗归因和可比实验。

Layer A · 规则层
先保证「多样性」
按黑马 / 小众 / 信任作者 / 热门分配配额,大厂最多占几席,黑马一定有位置。
↓
Layer B · LLM 判断
再下「判断」
在均衡候选池里,按加权标准选出最值得学的 5 个,并给出有证据、有可学点的理由。

从榜单到判断，
每一步都有依据。

API 数据、规则候选池、LLM 选择与最终报告分阶段留痕，让结果能复核，也让下一轮改进有据可依。
